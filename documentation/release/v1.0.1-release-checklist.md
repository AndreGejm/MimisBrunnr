# V1.0.1 Release Checklist

Use this checklist before creating the `v1.0.1` tag or packaging an installer.

## Repository State

- [ ] Merge the verified release branch to `main` or create a dedicated
      `release/v1.0.1` branch.
- [ ] Confirm `package.json` version is `1.0.1`.
- [ ] Confirm no local changes:
      `git status --short --branch`
- [ ] Confirm `HEAD` is the intended release commit:
      `git rev-parse HEAD`
- [ ] Do not reuse `v1.0.0`; that tag already exists on origin.

## Clean-Clone Verification

Run from a fresh clone, not the development worktree:

```bash
corepack pnpm install --frozen-lockfile
corepack pnpm build
corepack pnpm test:e2e
corepack pnpm test:eval:retrieval
corepack pnpm docker:mcp:build
```

Expected results:

- build exits `0`
- e2e tests pass
- retrieval eval passes all fixtures
- Docker MCP image builds
- `node:sqlite` experimental warning may appear and is not a failure

## Installer Inputs

- [ ] Confirm Node `>=22.0.0`.
- [ ] Confirm Corepack can run `pnpm@10.7.0`.
- [ ] Confirm Docker Desktop is installed when Docker profiles are selected.
- [ ] Confirm Docker Model Runner or an Ollama-compatible endpoint exposes:
      `qwen3-coder`, `qwen3:4B-F16`,
      `docker.io/ai/qwen3-embedding:0.6B-F16`, and `qwen3-reranker`.
- [ ] Confirm Qdrant availability or document degraded retrieval mode.
- [ ] Confirm Python runtime dependencies for `runtimes/local_experts`.

## State And Secrets

- [ ] Create or select `MAB_DATA_ROOT`.
- [ ] Back up existing canonical vault, staging vault, SQLite state, auth config,
      and `state/tool-output`.
- [ ] Generate operator/session auth tokens outside git.
- [ ] Confirm generated tokens, registry files, revoked-token files, SQLite
      files, and tool-output files are ignored or stored outside the repo.

## Legal Gate

- [ ] Add a repository license before public distribution.
- [ ] Check bundled RTK, Superpowers, Aider, Docker, model, and vendored runtime
      license obligations.

## Tagging

Only after all required checks pass:

```bash
git tag -a v1.0.1 -m "mimir v1.0.1"
git push origin v1.0.1
```
